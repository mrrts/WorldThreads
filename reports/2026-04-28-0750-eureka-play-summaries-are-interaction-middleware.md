# /eureka — /play summaries can become interaction middleware

*Generated 2026-04-28 07:50. This run opened after the copy-middleware eureka to
test whether a non-copy surface showed the same threshold-crossing pattern, or
whether copy work was special. The Focus-mode iteration chain answered yes, but
with a different category.*

## The discovery

In UI iteration arcs, `/play` summaries can cross from **stored evidence** into
**interaction middleware**.

This is not the same thing as copy middleware.

Copy middleware steers the next sentence.

Interaction middleware steers the next control-shape move.

Sometimes it also steers the stopping rule for the loop itself.

## Why this became visible

The Focus-mode chain is the worked example:

1. v1 verdict compresses to one seam: the all-or-nothing mode switch breaks
   active-writing flow
2. next move ships Context Peek
3. v2 verdict compresses to one seam: quick glances are cheaper, but sustained
   cross-reference still carries hold fatigue and state loss
4. next move ships quick-lock
5. v3 verdict compresses to one seam plus a stopping heuristic: converge on a
   single-sentence control scheme
6. next move ships v4 explicitly against that rule
7. v4 verdict refines the rule itself: expressibility alone was not enough;
   semantic uniformity across states also had to hold

That is not just evidence trailing behind implementation.

That is a report-family actively steering the next interaction pass.

## Why this is a different category from copy middleware

The Sapphire arc taught a sentence-shaped pattern:

- this line spends trust too fast
- ship B, not A
- this remaining phrase still leans on the reader

The Focus arc taught a state-shaped pattern:

- this control scheme still amputates required context
- this gesture trades one tax for another
- this loop stops only when the semantics are uniform across states

In both cases the summary moves in front of the next edit.

But what it steers is different.

In copy work, the summary becomes a live constraint on wording.

In interaction work, the summary becomes a live constraint on control shape,
state model, or stopping-rule logic.

## Why the stopping-rule refinement matters

The strongest evidence in the Focus chain is not just that one report led to
one implementation.

It is that the report-family eventually started steering the criterion by which
the loop would end.

First:

- stop when the interaction can be expressed in a single short sentence

Then:

- that rule was too weak; semantic uniformity across states must also hold

Once a report is shaping the loop's own termination criterion, it is plainly no
longer just archive.

It is operating as middleware for iteration.

## Scope

This category should stay narrow.

Not every `/play` report becomes interaction middleware.

The threshold-crossing pattern appears when:

- the run belongs to a live UI or control-scheme iteration loop
- the verdict compresses into one named state/flow seam
- the next move is an implementation pass on that exact seam
- later reports keep steering the loop through refined seam summaries or
  stopping rules

Absent that pattern, the report remains evidence only.

## Reading

The project now has three sibling precomposition categories with different
steering objects:

- `mission-arc` as **trajectory middleware** for reports and choosers
- seam-compressed `/play` summaries as **copy middleware** for wording arcs
- iteration-compressed `/play` summaries as **interaction middleware** for UI
  control-shape arcs

That is a useful deepening because it keeps the category honest.

The project does not need a vague doctrine that "summaries steer things."

It now has a sharper picture:

- some summaries steer prose
- some summaries steer interaction
- some trajectory reads steer whole composition acts before they begin

The Focus chain earned the third category by showing a report-family that did
not merely criticize and bless implementation, but repeatedly named the next
control-shape move and eventually refined the loop's own stopping rule.
