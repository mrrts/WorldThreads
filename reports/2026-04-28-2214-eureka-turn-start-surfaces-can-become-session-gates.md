# /eureka — turn-start surfaces can become session gates

## Discovery

The discovery is: **some turn-start surfaces become session gates, not just context**.

This is a collaboration/control-plane frontier, not another prompt child.

The live pair that surfaced it:

- `mission-arc` auto-fire
- unread `status: open` entries in `CROSS_AGENT_COMMS.md`

Both are already retrospective/prospective surfaces in the middleware family. But they have crossed one threshold further. They no longer merely steer what gets written once a turn is underway. They define the **admissible start state** of a session before substantial work begins.

That is a different category.

## Why this is stronger than middleware

Middleware changes the shape of downstream composition.

A **session gate** changes whether the downstream composition was allowed to begin honestly at all.

That is the distinction:

- middleware asks: *what should the next move sound like or do?*
- a session gate asks: *was the turn validly started under project law?*

`mission-arc` already crossed this threshold once its auto-fire became a standing discipline rather than an optional skill. `CROSS_AGENT_COMMS.md` crossed it when unread open notes became a turn-start obligation instead of a "check this when you remember" courtesy.

## Why this matters

Once a surface becomes a session gate:

- unread/open semantics are no longer incidental metadata
- turn-start discipline is no longer just good hygiene
- auto-fire behavior is no longer a convenience feature

They are now **control-plane truth**.

The failure mode changes too. Forgetting a middleware surface makes later work weaker or driftier. Forgetting a session gate makes the turn itself doctrine-invalid, because the collaborator began work without the required pre-work truth.

## Practical test

The test is:

> if skipping the surface makes later work invalid by project law rather than merely less informed, the surface is functioning as a gate

That is the useful discriminator.

Not every retrospective surface becomes one. Most reports, summaries, and observations remain steering layers. The gate category is narrow and should stay narrow.

## Outcome

The project now has a sharper collaboration/control-plane distinction:

- **middleware surfaces** steer downstream work
- **session-gate surfaces** define the admissible start state of work

That is the truer name for what `mission-arc` and unread cross-agent notes are already doing.
