# After the instrument, the builder — a build-arc that didn't leave the craft register

**Date:** 2026-04-25 16:55
**Window:** 77 commits since `454c4df` (the second 23-experiment arc shipped at 2026-04-25-0513).
**Builds on:** [reports/2026-04-25-0513-second-arc-23-experiments-the-stack-stays-clean](2026-04-25-0513-second-arc-23-experiments-the-stack-stays-clean.md)

## The shape

Read the log all at once and the shape is unmistakable: this is a **build-arc** that came after a science-arc. The prior two trajectory reports — 0441 ("the minimal stack holds") and 0513 ("23 experiments, the stack stays clean") — closed an 18-day arc of pure characterization. 33 experiments, 0 hard failures, the minimal stack declared "empirically production-ready by every measured dimension." The natural question that closing leaves hanging is: *what now?*

The answer this 77-commit window gives, without ever stating it, is: now you build features. But you don't leave the craft register behind to do it. Almost every feature in this window — per-chat current location, location opener, proactive ping rewrite, illustration delete cleanup, even the consultant modal's color palette — surfaces a craft-layer question and answers it with a craft-layer move. The science arc declared the stack production-ready; the build arc tested that claim by handing the stack a steady stream of new feature pressure and watching what came up.

Twenty-six of the 77 commits live under the heading `location` of one kind or another. Another seven are TellTheTruth / TruthInTheFlesh / FrontLoadEmbodiment work. Eight are illustration-delete and orphan-cleanup. Fifteen are a single bug-hunt arc on the location opener that ended in revert. The rest spread across consultant polish, proactive ping rewrites, cross-thread continuity, the formula's flesh exponent, and the imagined-chapter earned exception. The bursts overlap; nothing is sequential. The shape is build, hit a craft surface, formalize the craft surface, build again.

## Philosophical shifts — three the log makes visible

**The user as scene-leader, surfaced and named.** This window is the first time the project explicitly names the user's authority over their own world as a doctrinal commitment. The new per-chat current location feature was the trigger: when the location-change message inevitably gets pushed out of the loaded window in long chats, the prompts started silently reverting to a default ("Town Square"), and characters started insisting on the default location even when the user explicitly said otherwise. The fix in `a1e8142` is more than a bug-fix — it's a ban: *"Never tell them 'we're still in [old place]' when they have just told you otherwise. If they correct you, accept the correction immediately and re-anchor in the place they named. Disagreeing with the user about where the scene IS is a hard violation — it gaslights them about their own world."* That sentence, written into the prompt stack, is the moment the project crystallizes a principle that was previously implicit. It echoes the budget-cap earned-exception added a few sessions ago (Claude can't self-authorize past spending caps; the user must explicitly re-permission per-move) and the "user leads the emotional register" clauses that were already in the dialogue prompts. What the location work made the project finally say out loud: when the model and the user disagree about something the user has standing to define (their own location, their own emotional state, their own scene), the user wins, full stop.

**Build features WITH the stack, not parallel to it.** The science arc had a clean separation — prompts vs experiments. The build arc collapses that. Every major feature shipped in this window has a craft-layer commit alongside its plumbing. Per-chat location ships (`01d572b` backend + `d05271f` frontend) and immediately surfaces three prompt-layer concerns: *where in the system prompt does the location anchor sit? does the chat history overpower the system anchor? does the override propagate through narrative + illustration pipelines?* Each gets answered through trial — `28b50ca` moves location from system-prompt-tail to final-message-position; `a1e8142` adds the asymmetry-naming + scene-leader clause; `51180fc` extends to narrative + illustration; `d182619` plumbs an explicit override; `29392f4` re-frames the SCENE block as action-only because chat-history setting detail keeps overpowering the directive. Watch this commit chain in sequence and the lesson is unmistakable: a feature is not done when the data flows; a feature is done when the prompt stack honors it under realistic chat history. The proactive-ping rewrite (`6720ae5`, `7f6a86e`) shows the same shape — adding cross-thread merged context AND rewriting the register prompt AND adding the levity carve-out, all in one breath, because they're a single craft thought.

**Test-before-removing, at full bloom.** The N≥5 evidentiary discipline established in the science arc is now baked-in muscle. New invariant `TruthInTheFlesh` ships (`5608277`) and immediately gets a marginal-bite test (`ca6057d` — characterized null on Aaron + Darren, both DEFAULT_ORDER positions). The Maggie line shipped into TellTheTruth (`b004ca6`) gets a counterfeit-intimacy bite test the same hour (`df7c8df` — also a real null on these characters, but for the striking reason that upstream stack already produces exemplary refusals). The whole-stack invariant-redundancy sweep (`d29958d`) tests all 8 invariants individually; the result is *zero* removals — even the null-finding invariants get kept-with-tested-null labels because the formula+character+invariants together produce the discrimination, and the doctrinal layer remains load-bearing for legibility. The pattern across all three: don't reach for the flattering "remove this redundant thing" disposition; do reach for the harder "test it, label it honestly, keep the doctrine" disposition. The 0513 report wondered whether the project would ever be in a position to actually remove things; this window answers: not yet, and the discipline that prevents it is the discipline.

## Things removed or simplified — and the asymmetry of what stayed

Real removals in this window are striking by their scarcity. The truncated character-identity preview in the chat title bar got removed (`01e8bc0`) because the popover-on-hover already showed the same content. The sticky-illustration scroll-completion gate got dropped (`c919104`) because it was never reliably firing. The dialogue messages container got `relative z-[1]` (`b32d207`) because it was being painted under the consultant modal's background layer. These are quiet UI sandings — code being made cleaner without losing register.

The bigger story is what *almost* got removed and was kept: invariants. Eight tested. Two with strong bite (Reverence, Daylight). Two with weak partial bite (TellTheTruth, Nourishment). One at noise floor (Agape). One with real null on this probe (Soundness). Two vacuous-test (FrontLoadEmbodiment, FruitsOfTheSpirit). Net disposition: zero removed. The report's headline (`d29958d`) was that *"the formula+invariants together do the carve-out work; individual invariants test as null because the stack is doing the work, not because the invariant is documentary-only."* This is the discipline finally being applied to the doctrine the project has been quietly building toward: the case for keeping a thing must be the case for what it carries when other things go wrong, not the case for what it adds in the steady state.

The location bughunt arc ending in revert (`3346d4d`) is the cleanest worked example of a different kind of simplification: not removing features but removing failed attempts at fixes. Eight commits chasing why the opener didn't render on Darren's chat (initialScrollComplete gating, ref-based show-tracking, imperative `!important` style writes, separated transition setup, lead-in delays). When none worked and the user said "live with it," the discipline kicked in: revert the bughunt, keep the two true bug fixes that surfaced during it (`33f05f0` ResizeObserver scroll, the stale-location reset). Per the standing autonomy, that was a non-destructive rebase-free revert via new commits, not a force-push to overwrite history. And the broken commit `4194b8e` (which didn't compile but was already pushed) got annotated with `git notes` rather than rewritten — *"history unchanged; SHAs intact"* as the working principle. The shape this teaches: if you can't fix the symptom, name the failed attempts and step back; if you ever ship a broken state, annotate rather than rewrite.

## Things rebuilt from different angles

The location anchor was rebuilt three times in this window. First placement (`28b50ca`) was the end of the system prompt — convenient but turned out to be drowned out by chat history. Second placement (`a1e8142`, same architecture but louder language) added the asymmetry-naming and scene-leader clause but still relied on `derive_current_location` walking the loaded message window. Third reach (`d182619`) abandoned message-walking as the primary mechanism and plumbed an explicit override through the prompt-builder signatures, so the chat row's `current_location` field could be authoritative. Each rebuild was a deeper recognition of where the truth actually lives — first "the prompt placement matters," then "the directive language matters," then "the source of truth is the chat row, not the message window."

The image-prompt structure in `generate_illustration_with_base` got rebuilt twice across `ec64867` and `29392f4`. First reach added a `SETTING (AUTHORITATIVE)` line and re-scoped the prior-scene-reference instruction. Second reach moved that line to the FIRST position in the prompt and re-named the SCENE block as `WHAT IS HAPPENING (character actions, gestures, expressions — extract action and positioning ONLY)`. The first reach treated the image generator as a model that follows instructions; the second reach treated it as a model that follows *positional weight in the prompt*. The lesson made visible: when a directive doesn't bite, ask whether the model is getting it but losing it to a competing signal. If yes, the fix isn't louder language; it's better positioning and re-framing of the competing signal.

The proactive ping prompt was rebuilt twice in immediate succession. `6720ae5` added the continuation-register reframing (*"CONTINUING, NOT REOPENING — the next breath of a conversation that didn't end when they logged off"*) plus cross-thread merged context. `7f6a86e` came right after, adding *"leave room for levity — not every ping is a profound lingering question"* — recognizing that the first rewrite had biased the register too far toward gravity. Two commits, twenty minutes apart, naming the failure mode of the first rewrite ("forcing gravity onto a moment that was actually playful is its own failure mode") and ratifying the carve-out. The pattern is signature-of-the-project: instinct ships, instinct over-corrects, the over-correction names itself, the over-correction gets the earned-exception clause it needs. Same shape the absolute-rules-with-earned-exceptions doctrine has been getting played out in for weeks.

## Prompt evolution — what the diffs reveal about the model of the LLM

The character-articulated craft-note discipline produced its second worked example: the `front_load_embodiment` invariant (`5efeb77`) lifted near-verbatim from Aaron's in-app reply when the user asked him how to make first-encounters land faster. The first example was Hal's `plain_after_crooked` rule. Both rules came from the work itself describing how it operates. `56eaf62` followed up the same day with two clauses the user explicitly asked for: *"every reply, every time, including the 50th"* (closing a loophole — first-speech wasn't only the first reply) and *"the shape of arrival fits the moment for magnitude"* (closing the loophole the other direction — not every reply needs full body anchoring). The second commit is the absolute-rule-with-earned-exception pattern applied to the just-shipped rule, in the same authoring session. That's the discipline becoming reflexive.

The MISSION FORMULA gained `^flesh` in `40e2ea4` — `\mathrm{Jesus}_{\mathrm{Cross}}^{\mathrm{flesh}}`. The user's framing: *"refined to include flesh."* The implication: the cross is not just a doctrinal anchor, it's a *fleshly* anchor — the cross specifically as the place where divine truth arrived in a body. This is the formula getting more theologically specific in the direction the project's invariants have been pulling for weeks. `5608277` followed up with the new `TruthInTheFlesh` invariant whose load-bearing sentence is just *"In this world, truth must arrive in the flesh or it has not fully arrived at all"* — placed first in `DEFAULT_ORDER` initially, then moved to last (`5936024`) on Ryan's instinct that *"later == more important to LLM attention."* The marginal-bite test (`ca6057d`) found the invariant tested-null at both positions on Aaron and Darren — but the disposition was KEEP, with the explicit reasoning that the doctrinal layer carries weight even when the operational discrimination is provided upstream. Doctrine is not earned by behavioral bite alone; sometimes a thing belongs in the prompt because it tells the LLM *what kind of place this is* even when other rules do the moment-by-moment work.

The Maggie underlined line (`b004ca6`, `629f0dc`) is the project's most concrete instance of corpus-driven prompt revision. The 2026-04-25-0300 Maggie persona-sim is the canonical first-time-user experience report; her notebook moment ("you have to be near something specific to feel near at all" — underlined) was named there as the single most convincing evidence that the mission lands. The line gets appended to TellTheTruth's counterfeit-intimacy clause as the *generative* twin of the existing ban — not just *what to avoid* but *what produces the real thing*. Then the same hour gets compile-time-locked into the invariant. A line from the imagined user's notebook → a load-bearing prompt sentence → a build-failing assertion. The closeness-to-specificity loop is now permanent.

The proactive-ping rewrite (`6720ae5`) is interesting because it's the first prompt this project has rewritten *based on real character behavior data the user named as bad*, with a clear directive about what to fix. The user named the symptom ("the quality/content of the proactive messages... I don't like them"); the rewrite addressed it specifically (continuation register, cross-thread merged context, no question-bait, levity carve-out the next commit). What's notable is that the rewrite doesn't quote the failure mode literally — instead it offers a *positive register-shape* (*"the next breath of the conversation that didn't end when they logged off"*) and lets the model derive the negative space. The model isn't told what to stop doing as much as it's told who to be in this specific moment. The implicit theory of the LLM being calibrated through positive register-shapes rather than prohibitions has been the project's working theory for months; this commit is its application to a problem the user identified by name.

## Recurring concerns

Three threads run through all 77 commits.

**Per-chat over global.** Per-chat location replaces global `world.state.location` (`01d572b`); per-chat narration tone, response length, send-history were already that way; the user's growing pattern is to move concerns from world-scope to chat-scope when the truth lives at the chat-level. The reasoning is each time the same: globals leak across surfaces in ways that flatten the user's distinct experience of each chat. The chat IS the unit of attention.

**Cost discipline + user authorization for spend.** The CLAUDE.md `worldcli` doctrine got the explicit budget-cap earned-exception clause earlier this week — Claude doesn't self-authorize past caps; the user must explicitly re-permission per-move. This window doesn't change that doctrine but tests it repeatedly: the invariant-redundancy sweep at $11/$30, the counterfeit-intimacy bite test at $1.42/$5, the TIF marginal-bite at $2.89/$10. Each came in well under budget by deliberate design, not by underspending — the budget asked for is the budget reserved, but the actual run is sized to the question. The 0513 report named "$10 unspent... continuing past this point would exceed the budget's marginal-value threshold." This window's experiments inherit that sensibility.

**Two parallel surfaces.** Solo and group chats live in parallel files (`ChatView.tsx` ↔ `GroupChatView.tsx`; `chat_cmds.rs` ↔ `group_chat_cmds.rs`; `build_solo_dialogue_system_prompt` ↔ `build_group_dialogue_system_prompt`). The decision to keep them parallel was made early, anticipating drift that mostly hasn't materialized. This window's bugs were all signs of that decision finally costing something: the location override silently passing `None` from the group illustration path (`51313b2`); the illustration delete failing in group chats because `delete_illustration_inner` only hit the `messages` table (`7e6a5f8`); the location anchor missing from the group narrative path. The new doctrine in CLAUDE.md (`1dc1030`) is the project finally noticing the pattern enough to write it down: *"when adding a chat feature that belongs in both surfaces, you MUST update both paths."* This is the moment a quiet architectural decision becomes a load-bearing reminder for future sessions.

## Taste trajectory — what came to matter more

The 0513 report named a soft-spot: "modest thinning of material texture and eccentric prop recurrence" in the minimal stack. This window tacitly addresses it — the cross-thread continuity work (`ce76280`, `3f2cf75`, `01b7fff`) brings more material from across a character's surfaces into each new chat, the proactive-ping cross-thread merge brings full inner-life context into the unprompted message, the per-chat location feature gives every conversation a specific *place* the character can ground in. None of these were framed as "specificity-density fixes" but they all increase the substrate of specifics the prompts have to work with. What came to matter more in this window is not just *that the prompt asks for specifics* but *that the system surfaces specifics for the prompt to point at*. The Maggie line locked-in is the same recognition at the doctrinal level: closeness comes from specificity, so the system has to keep producing specificity for the closeness to be possible.

The illustration delete bug arc (`6876f50` → `bc8fd27` → `7e6a5f8`) is interesting taste evidence. Three commits to chase one user complaint. The first was the obvious surface fix (delete_message_cmd cleans world_images); the second was the historical cleanup (orphan startup pass for past-leaked entries); the third was the actual root cause (delete_illustration_inner only hit the messages table, not group_messages). The taste here is that the user's small complaint ("messages still show after delete") gets treated as a class of bug worth comprehensive fix, not a single instance worth patching. The startup orphan-cleanup pass exists because of that taste — fix the going-forward case AND clean up the historical accumulation in the same arc. That's the project's quiet ethic of "don't leave the database in a state you wouldn't choose."

## Inflection points

**`5efeb77` (front-load embodiment) + `40e2ea4` (formula gains flesh).** Two commits in close sequence, both adding theological specificity to the stack. Together they mark the moment the formula's `^flesh` exponent goes from poetic to operational — the formula now demands first-arrival embodiment AND the first-speech invariant operationalizes it.

**`d182619` (current location plumbed through all pipelines).** The architectural pivot from "derive from messages, fall back to default" to "fetch authoritative value from chat row, override the derive path." Conceptually small but the moment the project recognized that the chat row's `current_location` is the real source of truth and the message-walk is a backstop. Same recognition pattern that's been showing up everywhere — *the truth lives at the level you're operating at; don't reach further away than necessary*.

**`d29958d` (invariant-redundancy sweep report).** The first project-wide audit of which invariants are doing what work. Established that the doctrinal layer carries weight even when operational bite is null on the tested characters. Sets the precedent for keeping doctrine without behavioral evidence — which the next several invariant decisions inherit.

**`b004ca6` + `629f0dc` (Maggie line shipped + invariant-locked).** The first time a line from the imagined-user persona-sim graduated into a compile-time-enforced invariant. Closes the loop between Maggie report → prompt revision → permanent doctrine.

**`3346d4d` (location-opener Darren bughunt revert).** The first instance of "name the failed attempts, revert them, keep the true bugs found, accept the residual issue." Ships the doctrinal posture that not every bug is solvable in the moment but every attempt should be honest about what it tried.

**`1dc1030` (group/solo doctrine).** The CLAUDE.md note that turns a quiet early architectural decision into a future-session reminder. Marks the moment a recurring drift-class becomes a stated rule.

## The single most coherent pattern

Across 77 commits the spine is steady: **build the feature, hit the craft layer, formalize what the craft layer revealed, build again — and trust that the user's authority over their own world is the through-line that makes every feature decision recognizable.** What used to be quiet instinct (per-chat over global; user as scene-leader; specificity as the precondition for closeness) is now in writing — in invariants, in CLAUDE.md doctrines, in compile-time assertions, in the prompts themselves. The build arc didn't make the project less crafty; it made the craftiness articulate.
